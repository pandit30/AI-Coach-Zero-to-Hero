## The Key Idea

A pre-trained language model is trained to predict the next token — it's a completion machine. Ask it "What's the capital of France?" and it might respond "What's the capital of Germany?" because that's what comes next in a quiz format. Instruction tuning is the training step that transforms a completion machine into a model that actually follows instructions. It's how base models become assistants.

---

## The Gap Between Pre-Training and Following Instructions

Base model behaviour (no instruction tuning):

```
Prompt: "Translate the following English text to Hindi: Hello, how are you?"

Base model response (completion-style):
"Translate the following English text to Hindi: Goodbye, see you later?
Translate the following English text to Hindi: I am fine, thank you.
Translate the following English text to Hindi: ..."

(It continues the document format — doesn't actually translate)
```

After instruction tuning:

```
Same prompt → Model responds:
"नमस्ते, आप कैसे हैं?"

(Actually translates as instructed)
```

The knowledge to translate was always in the pre-trained model. Instruction tuning teaches the model *how to use that knowledge in response to commands*.

## How Instruction Tuning Works

Instruction tuning is supervised fine-tuning on a specific type of dataset: input-output pairs where the input is a human instruction and the output is the desired response.

```
┌──────────────────────────────────────────────────────────────────────┐
│              INSTRUCTION TUNING DATASET FORMAT                       │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Example 1:                                                          │
│  INSTRUCTION: "Summarize this text in 2 sentences."                 │
│  INPUT: [long article about quarterly results]                      │
│  OUTPUT: "ECHO India's Q3 revenue grew 23%... [2-sentence summary]" │
│                                                                      │
│  Example 2:                                                          │
│  INSTRUCTION: "Write a Python function that..."                     │
│  INPUT: [function requirements]                                     │
│  OUTPUT: [correct Python code]                                      │
│                                                                      │
│  Example 3:                                                          │
│  INSTRUCTION: "Is the following statement harmful? Explain."        │
│  INPUT: [statement]                                                 │
│  OUTPUT: [yes/no + explanation]                                     │
│                                                                      │
│  Train across thousands of such examples → model learns the        │
│  meta-skill of following instructions                               │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

## Key Instruction Tuning Datasets

**FLAN (Finetuned Language Net — Google):** 1,836 diverse NLP tasks reformatted as instructions. The landmark dataset that proved instruction tuning at scale.

**Alpaca (Stanford):** 52,000 instruction-following examples generated by GPT-3 (self-instruct technique). Showed that LLM-generated data can fine-tune a model cheaply.

**OpenAssistant:** 161,000 human-written conversation examples — real humans writing instructions and high-quality responses.

**ShareGPT:** Millions of real ChatGPT conversations shared by users. Trains on actual human-AI interaction patterns.

**Custom domain instruction datasets:** What ECHO India would create for HR — thousands of HR-specific instruction-response pairs.

## The Self-Instruct Method — Scaling Data Generation

Creating instruction datasets by hand is slow. The self-instruct method (Wang et al., 2022) uses an existing model to generate the dataset:

```
┌──────────────────────────────────────────────────────────────────────┐
│              SELF-INSTRUCT PIPELINE                                   │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. Start with 175 seed instructions (written by humans)            │
│                                                                      │
│  2. Ask a strong model (GPT-4) to generate more instructions:      │
│     "Generate 5 more diverse instructions similar to these:"       │
│     [seed examples]                                                 │
│                                                                      │
│  3. Filter low-quality instructions (duplicates, trivial ones)     │
│                                                                      │
│  4. Ask the model to generate an input and response for each       │
│     new instruction                                                 │
│                                                                      │
│  5. Filter and validate the responses                               │
│                                                                      │
│  6. Add to training dataset → use to fine-tune a smaller model    │
│                                                                      │
│  Result: 52,000 instruction-response pairs generated at ~$600      │
│  (cost of GPT-3 API calls to generate). Stanford Alpaca used this. │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

For ECHO India: generate thousands of HR instruction-response pairs using Claude or GPT-4, then fine-tune a smaller, cheaper model with them. One-time generation cost; permanent capability gain.

## Instruction Tuning vs RLHF — What's the Difference?

This confuses many people. Here's the distinction:

**Instruction tuning:** Supervised. You show the model correct input-output pairs. The model learns to imitate these examples. It learns what good responses look like for various instructions.

**RLHF (next session):** Reinforcement learning. You don't just show examples — you have humans compare two responses and say which is better. A reward model is trained on these preferences, then used to optimise the model via RL. This goes beyond "imitate examples" to "learn what humans prefer."

Most production models go through both: instruction tuning first (to make them follow instructions), then RLHF (to make them helpful, harmless, and honest).

## Instruction Tuning for Domain Adaptation at ECHO India

Custom instruction tuning pipeline for ECHO India's HR assistant:

1. **Generate base dataset:** Use Claude to create 1,000 HR-specific instruction-response pairs across categories: leave queries, policy questions, ticket classification, report generation
2. **Human review:** Domain experts review and correct 200 examples — these become the "gold" training set
3. **Self-instruct expansion:** Use Claude to generate 5,000 more examples similar to the gold set
4. **Fine-tune with LoRA:** Apply to a 7B model using QLoRA on 1 A100 (cost: ~₹15,000)
5. **Evaluate:** Test against 100 held-out examples, measure task accuracy

Result: An HR-assistant model that reliably follows ECHO India-specific instructions and uses ECHO India terminology correctly.

## Key Takeaway

Instruction tuning transforms a completion-machine (base model) into an instruction-follower (assistant). It's supervised fine-tuning on instruction-response pairs — the model learns the meta-skill of following directions.

The self-instruct method scales dataset creation: use a strong model to generate thousands of instruction pairs, filter and fine-tune a smaller model. Cost-effective and surprisingly high quality.

Instruction tuning is Step 1 in making a model useful. RLHF (next session) is Step 2: making it genuinely helpful and aligned with what humans prefer.
